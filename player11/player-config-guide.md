# 玩家移动系统 - 配置说明

---

## 一、文件结构

与你角色移动相关的文件有三个：

| 文件 | 作用 |
|------|------|
| `player11\player.tscn` | 玩家场景文件，存放角色节点结构 |
| `player11\player.gd` | 玩家脚本文件，控制移动和动画逻辑 |
| `tool\ueisukun_sprite.png` | 精灵图（角色贴图），8列×8行的精灵表 |

你不用管代码是怎么写的，本说明只告诉你 **在哪里调参数**。

---

## 二、操作步骤

### 步骤1：给角色添加碰撞体（必须做）

`move_and_slide()` 是 Godot 内置的移动方法，但它需要一个"碰撞体"来确定角色的物理边界。没有碰撞体角色也能动，但无法和其他物体（墙壁、敌人等）碰撞。

**操作方法：**

1. 在 Godot 编辑器中，双击打开 `player11\player.tscn`
2. 在左侧"场景"面板中，**右键点击 `player` 节点** → 选择"添加子节点"
3. 搜索 `CollisionShape2D`，选中后点击"创建"
4. 在右侧"属性检查器"面板中，找到 **Shape** 属性，点击"<空>"旁边的下拉框
5. 选择"新建 CircleShape2D"（圆形碰撞体）
6. 点击刚才创建的 CircleShape2D，在属性中找到 **Radius（半径）**，设为 `32`

> **提示**：你可以根据需要调整半径大小，半径越大角色越难穿过窄缝，半径越小越灵活。

### 步骤2：调整精灵位置（可选）

你的精灵图当前偏移是 `(1, -103)`，这是为了让图像在碰撞体上方显示。如果角色看起来位置不对，选中 `Sprite2D` 节点调整它的 **Position** 属性即可。

## 三、可调参数详解

选中 `player` 节点（不是 Sprite2D），在右侧属性面板的"脚本变量"分类下可以看到所有参数：

### 移动相关参数

| 参数名 | 默认值 | 作用 | 推荐范围 |
|--------|--------|------|----------|
| `move_speed` | 200.0 | 角色最快能跑多快（像素/秒） | 100 ~ 400 |
| `acceleration` | 1200.0 | 按下按键后，角色多快加到全速 | 500 ~ 2000 |
| `friction` | 1000.0 | 松开按键后，角色多快停下来 | 500 ~ 2000 |

**调参技巧：**
- 想要**更灵敏的手感**：同时加大 `acceleration` 和 `friction`（比如都设成 1500）
- 想要**带惯性/滑冰感**：减小 `friction`（比如设成 400）
- 想要**坦克般稳重的感觉**：减小 `acceleration`，加大 `friction`

### 动画相关参数

你的精灵图是 8列×8行 的表格，你需要在精灵图中观察：
- 第 0 行到第 7 行，哪些行走的是哪个方向的动画
- 每行有几帧是走路动画

| 参数名 | 默认值 | 作用 |
|--------|--------|------|
| `row_down` | 0 | 按 S（下）时显示精灵图第几行 |
| `row_left` | 1 | 按 A（左）时显示精灵图第几行 |
| `row_right` | 2 | 按 D（右）时显示精灵图第几行 |
| `row_up` | 3 | 按 W（上）时显示精灵图第几行 |
| `walk_frame_start` | 0 | 走路动画从第几列开始 |
| `walk_frame_end` | 2 | 走路动画到第几列结束（在这之间循环） |
| `idle_frame` | 0 | 站着不动时显示第几列 |
| `animation_frame_duration` | 0.15 | 动画每帧持续多少秒 |

**精灵表坐标说明：**

```
	  列0  列1  列2  列3  列4  列5  列6  列7
行0   [  ][  ][  ][  ][  ][  ][  ][  ]
行1   [  ][  ][  ][  ][  ][  ][  ][  ]
行2   [  ][  ][  ][  ][  ][  ][  ][  ]
行3   [  ][  ][  ][  ][  ][  ][  ][  ]
...
```

行列编号都从 **0** 开始。比如你的精灵图中：
- 如果角色朝下走的动画在精灵图**第1行（从上往下数第2行）**，那就把 `row_down` 改成 `1`
- 如果朝下走的动画只用了前3列（第0、1、2列），那就保持 `walk_frame_start=0`、`walk_frame_end=2`

> **重要**：`frame_coords` 的坐标是 `(列, 行)`，即先写列号再写行号。代码中已按这个格式处理，你只需要填数字即可。

## 四、在编辑器中查看精灵表

如果你想直观看到精灵表上每格的内容：

1. 在编辑器中选中 `Sprite2D` 节点
2. 在属性面板找到 **Animation** 分类
3. 调整 **Frame** 属性的数字，可以看到不同格子的画面
4. 或者直接在底部预览窗口中查看纹理的分割情况

## 五、常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 按WASD角色不动 | 缺少 CollisionShape2D | 按照"步骤1"添加碰撞体 |
| 角色动了但是画面不变 | 方向行号没配对 | 检查 `row_down` 等参数是否指向了精灵表正确的行 |
| 动画播放太快/太慢 | 帧间隔时间不对 | 调整 `animation_frame_duration`，变大=变慢，变小=变快 |
| 角色朝向不对（按W却面朝下） | 行号映射错误 | 确认 `row_up` 指向的是朝上的行 |
| 松开按键角色滑太远 | `friction` 太小 | 把 `friction` 调大，比如 1500 |
| 按下按键起跑太慢 | `acceleration` 太小 | 把 `acceleration` 调大，比如 2000 |
| 角色速度太慢/太快 | `move_speed` 不合适 | 调整 `move_speed` |
| 碰撞体和图像位置对不上 | Sprite2D 偏移不对 | 选中 Sprite2D 调整它的 Position |

## 六、参数实时调试技巧

Godot 编辑器支持**运行中实时调参**：

1. 按 F5 运行游戏
2. 切回 Godot 编辑器窗口（不要关掉游戏）
3. 在场景树中选中 `player` 节点
4. 修改右侧属性面板中的参数
5. 游戏中的角色会立即响应你的修改

这样你不需要反复停止/启动游戏来测试参数，改一个数字立刻看到效果。

## 七、按键映射

当前四个方向按键已配置好，你也可以在编辑器中修改：

- 菜单栏 → 项目 → 项目设置 → 输入映射
- `up` = W 键
- `down` = S 键
- `left` = A 键
- `right` = D 键

如果你要添加新的按键（比如冲刺 Shift、攻击 J），也在同一个地方添加。
